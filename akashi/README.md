# akashi-mcp

AKASHI / マネーフォワード クラウド勤怠Plus（クラウド勤怠管理システム）の MCP サーバー。公式 MCP 未提供。

## セットアップ

```bash
cd akashi
pip install -e .
```

## 環境変数

| 変数名 | 説明 |
|--------|------|
| `AKASHI_COMPANY_ID` | AKASHI 企業ID（ログイン URL から確認） |
| `AKASHI_TOKEN` | アクセストークン（管理画面 > 設定 > API設定で発行） |  # pragma: allowlist secret

## Claude Desktop 設定

```json
{
  "mcpServers": {
    "akashi": {
      "command": "akashi-mcp",
      "env": {
        "AKASHI_COMPANY_ID": "your_company_id",
        "AKASHI_TOKEN": "your_access_token_here"  # pragma: allowlist secret
      }
    }
  }
}
```

## 利用可能なツール

| ツール名 | 説明 |
|---------|------|
| `get_stamps` | 自身または指定した従業員1名の指定期間の打刻記録を取得する（最長6ヶ月・最大1000件、staff_id省略時は本人分） |
| `post_stamp` | 打刻を1件記録する（出勤・退勤・直行・直帰・休憩入・休憩戻）。呼び出すたびに新しい打刻イベントが追加される（べき等ではない） |
| `list_staffs` | 従業員一覧を取得する（1ページ最大20名、page は0始まり） |
| `get_staff` | 特定の従業員1名の詳細情報を取得する |
| `get_alerts` | トークンに紐づく従業員本人の勤怠アラート一覧を取得する（打刻忘れ・残業超過など、絞り込み条件なし） |

## 既知の制約

- **打刻の更新・削除ができない**: Akashi の公開APIには打刻を新規記録する API（`post_stamp` が使用）と取得する API（`get_stamps` が使用）しかなく、既存の打刻を修正・削除するAPIは提供されていない。誤って打刻した場合は管理画面から手動で修正する必要がある。
- **`post_stamp` はべき等ではない**: 同じ内容で複数回呼び出すと、その都度別の打刻イベントとして記録される（重複排除やマージは行われない）。
- **`get_stamps` は1回のリクエストにつき最大1000件・最長6ヶ月まで**: これを超える期間・件数を取得するには、期間を分割して複数回呼び出す必要がある。カーソル等によるページネーションは提供されていない。また、複数従業員をまとめて取得する `multiple_stamps` API（対象期間は最長1日・最大50名まで）が Akashi 側には存在するが、本コネクタでは未実装。
- **`list_staffs` のページ番号は0始まり**: Akashi API の仕様上、`page=0` が1〜20人目、`page=1` が21〜40人目を表す。
- **従業員の新規登録・更新・退職/削除は未実装**: Akashi API には従業員情報の新規登録・更新（PATCH）、退職・削除（DELETE）の API が存在するが、本コネクタは読み取り専用（`list_staffs` / `get_staff`）のみを実装しており、これらの書き込み系操作は提供していない。
- **`get_alerts` に絞り込みパラメータがない**: 日付範囲や従業員IDでの絞り込みはAPIとして提供されておらず、常にトークンに紐づく本人分の全アラートが返る。
